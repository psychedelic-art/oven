## Deliverables

### 1. Download handler

Streams the file bytes through the server with
`Content-Disposition: attachment`. Uses the storage adapter's
`getUrl(key)` + a `fetch()` proxy (Vercel Blob is public by URL,
so the server fetches its own blob URL and re-streams).

```ts
export async function GET(request: NextRequest, { params }: { params: { id: string } }) {
  const id = Number(params.id);
  if (!Number.isFinite(id)) return badRequest('Invalid file id');

  const callerTenantIds = await getTenantIdsFromRequest(request);
  const [row] = await db.select().from(files).where(eq(files.id, id)).limit(1);
  if (!row) return notFound();
  if (row.tenantId != null && !callerTenantIds.includes(row.tenantId)) return notFound();

  const res = await fetch(row.publicUrl);
  if (!res.ok || !res.body) return new NextResponse('Upstream failed', { status: 502 });

  return new NextResponse(res.body, {
    status: 200,
    headers: {
      'Content-Type': row.mimeType,
      'Content-Disposition': `attachment; filename="${encodeURIComponent(row.filename)}"`,
      'Content-Length': String(row.sizeBytes),
    },
  });
}
```

### 2. Public-serve handler

Serves bytes without authentication. The route is registered in
`apiEndpointPermissions` with `isPublic: true`. Only files whose
`visibility = 'public'` row are servable.

```ts
export async function GET(_request: NextRequest, { params }: { params: { pathname: string[] } }) {
  const storageKey = params.pathname.join('/');
  const [row] = await db.select().from(files).where(eq(files.storageKey, storageKey)).limit(1);
  if (!row || row.visibility !== 'public') return notFound();

  const res = await fetch(row.publicUrl);
  if (!res.ok || !res.body) return new NextResponse('Upstream failed', { status: 502 });

  return new NextResponse(res.body, {
    status: 200,
    headers: {
      'Content-Type': row.mimeType,
      'Cache-Control': 'public, max-age=3600',
      'Vary': 'Accept-Encoding',
    },
  });
}
```

## Risks

- **Graduation pressure causing acceptance-criteria shortcuts.**
  Mitigation: this sprint is a gate, not a feature. If any
  checklist item fails, do not advance — write a new
  `sprint-06-<topic>.md` and continue.
- **Download proxy performance** — streaming through the server
  adds latency vs. a direct Vercel Blob URL redirect. For private
  files the latency is unavoidable. For public files the
  dashboard can link directly to `publicUrl` and skip the proxy.
  This is documented, not fixed.
